"It is a profoundly erroneous truism, repeated by all copy-books and by eminent people when they are making speeches, that we should cultivate the habit of thinking of what we are doing. The precise opposite is the case. Civilization advances by extending the number of important operations which we can perform without thinking about them. Operations of thought are like cavalry charges in a battle — they are strictly limited in number, they require fresh horses, and must only be made at decisive moments."

— Alfred North Whitehead, 1911

Software has gotten incredibly cheap to build. You can just make one-off things for experiments, tasks, etc...

My monthly GitHub commits

Today I'll discuss general iteration loop with Claude Code and specific examples

  • Color ramp tuning with Palette
  • Synthetic training data with Trellis
  • Fitness flux workflow and figures
  • (If there's time) code decrufting and performance improvements in Nextstrain

General workflow

  • One project = one GitHub repo = one web Claude conversation thread
  • This thread keeps context for Claude in one place
  • I prefer web Claude for learning about a topic, its internet search is better than Claude Code and it's tuned towards more "blue skies" thinking than Claude Code's skilled plumbing
  • I produce "spec" documents in Markdown from web Claude and use these to plan out features in Claude Code
  • Development flow of spec → plan mode → implement plan → refine / fixups → commit
  • The spec document is committed alongside other commits in the feature branch / PR, serving to document design goals

Color ramp tuning with Palette

My original Nextstrain color ramp was a one off Mathematica notebook. In working on updated color ramp I wanted a tool for live tuning, but this is something that's incredibly bespoke and there's no available bit of software. However, it's now possible to just build the specific tool in a couple days of work.

  • Palette tuner at localhost:5173

Synthetic training data with Trellis

Lattice proteins are a particular scientific instrument that was familiar with from the literature, but there's not great simulators to accomplish what I wanted to accomplish (Jesse's repo from 2011 came decently close, but there were a number of things that I still wanted). Engineering effort was easy and we get all kinds of unit tests, etc... in a way that I'm always too lazy / too busy to implement.

Plus I get a bunch of coaching on how to submit large-scale AWS Batch jobs.

  • Demo trajectory at blab.github.io/trellis/viz/
  • GitHub repo at github.com/blab/trellis

Fitness flux workflow and figures

Although I've used Python for workflows for a while now, I was still relying on Mathematica for figures. Here, I moved to everything in workflow with Observable Plot as the viz workhorse. It's verbose, but Claude is good at verbose. This is a similar, but more atomic direction, to the Observable Plot dashboards you saw previously.

  • Online manuscript at blab.github.io/fitness-flux/
  • GitHub repo at github.com/blab/fitness-flux

Code decrufting and performance improvements in Nextstrain

Bioinformatic software is very often characterized by transforming some particular input files into some particular output files. Claude Code benefits when there's a stable harness to evaluate against and producing byte-identical output is a powerful constraint.

  • 2X speed up to augur filter subsampling in Augur PR #2018
  • Mainly code cleanup but some performance and features to ncov and ncov-ingest
